# ztraj Phase 1 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a high-performance MD trajectory analysis library + CLI in Zig with RMSD, distances, angles, dihedrals, Rg, center-of-mass, inertia tensor, H-bonds, contacts, and RDF.

**Architecture:** Flat structs + free functions. SOA coordinate layout for SIMD. Frame-at-a-time streaming for large trajectories. File parsers adapted from zsasa. All results validated against mdtraj.

**Tech Stack:** Zig 0.15.2, zxdrfile (XTC dependency), Nix flake for build

**Spec:** `docs/superpowers/specs/2026-03-17-ztraj-design.md`

**Reference codebase:** zsasa at `/Users/nagaet/freesasa-zig/`

- [ ] **Step 8: Implement FrameIterator interface**

Define the common streaming interface that XTC and DCD readers will conform to:

```zig
pub const FrameIterator = struct {
    // vtable-style dispatch for format-agnostic iteration
    nextFn: *const fn (*FrameIterator) ?*const Frame,
    resetFn: *const fn (*FrameIterator) void,
    deinitFn: *const fn (*FrameIterator) void,
    // Shared frame buffer (reused across calls to minimize allocation)
    frame: Frame,

    pub fn next(self: *FrameIterator) ?*const Frame { return self.nextFn(self); }
    pub fn reset(self: *FrameIterator) void { self.resetFn(self); }
    pub fn deinit(self: *FrameIterator) void { self.deinitFn(self); }
};
```

- [ ] **Step 2: Run test — should fail**

- [ ] **Step 3: Implement QCP RMSD**

Reference implementation: mdtraj's `_rmsd.pyx` and the Theobald 2005 paper.

Algorithm:
1. Center both coordinate sets (subtract center of mass)
2. Compute inner product matrix M = sum(x_ref * x_tgt, etc.)
3. Compute characteristic polynomial coefficients
4. Solve for largest eigenvalue via Newton-Raphson iteration
5. RMSD = sqrt(max(0, (G_ref + G_tgt - 2*lambda_max) / n_atoms))

```zig
pub fn compute(
    ref_x: []const f32, ref_y: []const f32, ref_z: []const f32,
```
